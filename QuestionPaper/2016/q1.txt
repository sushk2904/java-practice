//* In Java, both Abstract Classes and Interfaces are used to achieve abstraction (hiding the complex implementation details and showing only the essential features). However, they serve entirely different architectural purposes.

1. an abstract class is a restricted class that cannot be used to create objects. 
A class can extend only one abstract class.
It is designed to act as a blueprint for other classes, allowing you to define a common structure and shared behavior for a group of related subclasses.
Constructors are supported 
2. Use an Interface 
It is created when a class only has abstract methods.
Constructors are not supported
A class can implement multiple interfaces.
